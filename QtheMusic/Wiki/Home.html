QTheMusic

Abstract Description

The QTheMusic project is an attempt to revolutionize the average play list, making it interactive in a way that lets multiple devices contribute to the order in which the tracks are played.

Entity Description

The intended finished product essentially consists of three entities: A central server, a host application and a client application. The communication between the three entities happens over the internet using the TCP protocol.

The Central Server

The Central Server essentially serves as a lookup table or a DNS server. A host may register itself at the central server which makes a client able to find its address and port which is open to clients. A host will then remain connected to the central server until it is no longer hosting.

The Host

The Host has list of tracks which it makes available to all its clients. The clients connected to the host may then choose which tracks should be played by the host. This is done using a queue, each client may add tracks to the queue. Multiple clients may be connected to the host at a time. The Host needs to register and stay connected to The Central Server in order for it to act as a host.

The Client

The Client connects to a host in order to partake in the creation of the music queue. The Client can then choose what tracks will be played by the host by adding them to The Hosts queue.
